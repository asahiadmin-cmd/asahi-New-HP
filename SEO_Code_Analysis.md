# ソースコード修正を伴うSEO対策案（検討用）

ソースコード（`src/App.jsx` や `index.html` など）の改修を前提とした場合、さらに強力なテクニカルSEOを実装することが可能です。

※ご指定の通り、**ここでは検討のみとし、実際のコード変更・実行はまだ行いません。**

---

## 1. ページごとのURL化と動的メタタグ（React Router + Helmetの導入）
**【重要度：★★★★★】**

*   **現状の課題**: 現在は単一ページアプリケーション（SPA）として作られており、「会社概要」や「求人一覧」のボタンを押してもURLが変わりません（常に `https://asahi1.co.jp/`）。そのため、Googleは「求人ページ」や「会社概要ページ」を個別のページとして認識できず、ユーザーも求人ページを直接ブックマークしたりシェアしたりできません。
*   **修正案**:
    *   `react-router-dom` を導入し、`/recruit`（求人）、`/company`（会社概要）といった**個別のURLを発行**します。
    *   `react-helmet-async` を導入し、ページを切り替えた際にブラウザのタブ名（`<title>`）やメタディスクリプションが自動で切り替わるようにします（例：「求人一覧｜ライフサポートあさひ」など）。これにより、各ページが固有のキーワードで検索に引っかかりやすくなります。

## 2. セマンティックHTML（見出しタグの構造最適化）
**【重要度：★★★★☆】**

*   **現状の課題**: Reactのコンポーネント内（`App.jsx`）で、見出しタグ（`<h1>`, `<h2>`, `<h3>`）の階層構造が少し曖昧になっています。特に、ページ内で最も重要なテーマを示す **`<h1>` タグが適切に配置されていません**（現在はロゴが `<span>`、スライダーのキャッチコピーが `<h3>` になっています）。
*   **修正案**:
    *   トップページの一番目立つキャッチコピー、またはサイトのヘッダーロゴに `<h1>` を割り当てます（例：`<h1>東大阪市の訪問介護・ヘルパー求人ならライフサポートあさひ</h1>`）。
    *   各セクションの見出し（「数字で見るあさひ」「求人情報」など）を `<h2>` に統一し、その下の小見出しを `<h3>` にするというように、HTMLの構造をクローラーが理解しやすいように整理します。

## 3. 画像の最適化と `alt` 属性の強化
**【重要度：★★★☆☆】**

*   **現状の課題**: いくつかの画像（スタッフ写真など）がGoogleドライブから読み込まれています。また、画像の `alt`（代替テキスト）がシンプルなものになっています。
*   **修正案**:
    *   **画像のホスティング**: Googleドライブの画像URLは表示速度やクローラーの画像解析に不利になることがあるため、画像をプロジェクト内の `public` フォルダに保存するか、専用のサーバー/CDNから配信するように変更します。
    *   **alt属性の記述**: `<img alt="スタッフ">` ではなく、`<img alt="東大阪市で働く訪問介護スタッフの〇〇さん">` のように、検索キーワードを含めた具体的な説明文（alt属性）を追加します。
    *   **Lazy Loading**: 画面のスクロールに合わせて画像を読み込む `loading="lazy"` 属性を付与し、ページの初期表示速度（Core Web Vitals）を向上させます。

## 4. プリレンダリング（静的HTMLの生成）の導入
**【重要度：★★★☆☆】**

*   **現状の課題**: Vite + Reactで作られたSPAは、ブラウザ上でJavaScriptが実行されて初めて画面が描画されます。`index.html` に `<noscript>` フォールバックを追加したため対策はできていますが、本来のReactのDOMツリー自体はクローラーにとって読み込みに時間がかかる場合があります。
*   **修正案**:
    *   Viteのプラグイン（`vite-plugin-prerender` など）を導入し、ビルド時にあらかじめ完全なHTMLファイル（描画済みのDOM）を生成するよう設定します。これにより、初期表示速度が劇的に向上し、SEOの評価指標である「LCP (Largest Contentful Paint)」スコアが改善されます。

## 5. パフォーマンス・チューニング（コード分割）
**【重要度：★★☆☆☆】**

*   **現状の課題**: `App.jsx` が非常に長く（約2,000行）、すべてのコンポーネントが一度に読み込まれています。
*   **修正案**:
    *   Reactの `lazy` と `Suspense` を使って、各ページ（トップ、会社概要、求人など）のコードを分割します。これにより、ユーザーが最初にアクセスしたときにダウンロードするファイルサイズが小さくなり、スマートフォンのようなモバイル回線での表示速度がさらに速くなります（モバイルSEO対策）。
